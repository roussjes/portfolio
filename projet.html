Concevoir une base de données

Compétences utilisées :

  • Utilisation de requêtes SQL pour intérroger une base de données
  • Concevoir une base de données à partir d'un SEA, d'un SLR et de règles de gestion
  • Visualiser et comprendre des données

Détails du projet :

  • Projet réalisé en binôme
  • La durée du projet était de 1 mois
  • Nous avons rendu plusieurs fichier tout au long du projet

Lors de ce projet, Nous avons commencé par créer un schéma entité association, Le thème de notre projet était le crash du Titanic sur lequel nous avons donc fait une analyse afin de mieux comprendre les données que nous avons traités par la suite avec mon binôme. Ce schéma entité association était le premier rendu que nous avions à faire.
J'ai donc posé les règles de gestion (avec l’aide de contrainte donnée dans le sujet) afin de pouvoir créer ce schéma entité association de notre base de données.
En deuxième partie, j'ai dû justifier le schéma du modèle relationnel de notre base qui nous était donné dans le sujet en expliquant pourquoi une table possède cette clé primaire ou pourquoi possède t-elle cette cardinalité vers une entité précise.
Enfin comme dernière tâche, j’ai exprimé les contraintes de chaque table (contrainte d’attribut, de check, d’intégrité référentielle).
Cette deuxième et dernière partie à constituer notre deuxième et troisième rendu.

Installation d'un environnement de développement

Compétences utilisées :

  • Installer des packages d'applications
  • Installer un environnement graphique et un système d'exploitation
  • Visualiser et comprendre une liste d'étapes

Détails du projet :

  • Projet réalisé seul
  • La durée du projet était de quelques heures
  • J'ai rendu plusieurs captures d'écran à la fin du projet

Lors de ce projet, J'ai commencé par l'utilisation d'une machine virtuelle vierge, que j'ai par la suite rendu utilisable en installant un système d'exploitation afin de configurer par la suite le poste pour du développement. J'ai commencé par configurer la machine virtuelle en installant Debian (Linux) et l'environnement graphique KDE/Plasma.
La machine était destiné à un developpeur. J'ai donc commencé par installer GIT. puis l'IDE IntelliJ.
J'ai installer cette IDE de trois façon différente, En premier grâce à snap package puis grâce à flatpak package et enfin grâce à une archive.
Ce projet m'a permit de comprendre la configuration d'un système ainsi que l'installation de programme sur une machine Linux.

Classement automatique de dépêches à l'aide d'algorithmes en Java

Compétences utilisées :

  • Manipuler le language de programmation Java
  • Comprendre la compléxité d'un algorithme
  • Trier des mots afin d'obtenir le meilleur tri possible

Détails du projet :

  • Projet réalisé en binôme
  • La durée du projet était de 1 semaine
  • Nous avons rendu un rapport en anglais pour expliquer notre projet

Lors de ce projet, Nous avons commencés par créer des fichiers lexiques que nous avons progressivement rempli avec autant de mots liés au catégorie (demandées dans le sujet) que possible pour atteindre un pourcentage élevé de correspondance pour chaque dépêches (environ 80 mots par fichier).
Ensuite nous avons créer plusieurs fonctions pour générer automatiquement les lexiques afin de comparer les pourcentages de correspondance avec nos lexiques.
Ainsi que pour calculer un score de correspondance entre nos mots et leurs apparitions dans les dépêches.
Enfin j'ai fais l'optimisation de nos fonctions en analysant la complexité de nos codes pour que le classement se fasse plus vite (nous sommes passé d'une recherche séquentielle à une recherche binaire).
